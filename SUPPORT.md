# Support

Limelight‑X is an open, community‑driven project.  
There is no paid support and no sponsorship program — the best way to support the project right now is to use it, talk about it, and help it grow.

This document explains where to ask questions, how to get help, and how to contribute in non‑financial ways.

---

## Getting Help

If you have a question about Limelight‑X:

- **Check the README** for usage, examples, and project goals  
- **Search existing Issues** to see if your question has already been answered  
- **Open a new Issue** if you’re stuck or something is unclear  

Please include:

- what you were trying to do  
- the input you used  
- the output you expected  
- the output you actually got  

Clear, reproducible examples help everyone.

---

## How to Support the Project (Non‑Financial)

You can support Limelight‑X in meaningful ways without spending a cent:

### **1. Spread the word**
- Share the project with colleagues or communities  
- Write about your experience using it  
- Mention it in talks, blogs, or social posts  

Awareness is oxygen for early‑stage projects.

### **2. Give feedback**
- Tell us what works well  
- Tell us what’s confusing  
- Suggest improvements to the DSL, evaluator, or documentation  

Thoughtful feedback is incredibly valuable.

### **3. Contribute**
- Fix a bug  
- Improve documentation  
- Add examples  
- Propose enhancements  
- Help refine the grammar or IR  

See `CONTRIBUTING.md` for details.

### **4. File issues**
Bug reports, feature requests, and design discussions all help shape the project’s direction.
